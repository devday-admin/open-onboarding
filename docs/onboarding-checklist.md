# Developer Onboarding Checklist

Complete guide for your first day, week, and month using AI assistance.

## Day 1: Setup & Orientation

### Administrative
- [ ] HR paperwork complete
- [ ] Access to email, Slack, GitHub, etc.
- [ ] Development machine set up
- [ ] IDE/editor installed and configured

### AI Tool Setup
- [ ] Choose your AI tool (Claude Code, Cursor, or Cline)
- [ ] Install your chosen AI tool
- [ ] Copy first-week onboarding config
- [ ] Test config with simple questions
- [ ] Read configuration guide (if using Claude Code)

### Team Introductions
- [ ] Meet your manager/mentor
- [ ] Meet your immediate team
- [ ] Understand team structure
- [ ] Schedule 1:1s with key people

### Codebase Access
- [ ] Clone main repository
- [ ] Get dependencies installed
- [ ] Run project locally (even if errors - that's normal!)
- [ ] Verify you can access staging/dev environments

### Using AI for Day 1
**Try asking your AI:**
- "Explain the folder structure of this codebase"
- "Where should I start exploring as a new developer?"
- "What documentation should I read first?"

## Week 1: Learning & Exploration

### Codebase Understanding
- [ ] Read main README and getting started docs
- [ ] Understand project architecture (ask AI for overview)
- [ ] Run the project and explore main features
- [ ] Identify key components/modules
- [ ] Read tests to understand expected behavior

### Development Workflow
- [ ] Understand git workflow (branching strategy)
- [ ] Learn how to run tests
- [ ] Know how to run linter/formatter
- [ ] Understand code review process
- [ ] Learn deployment/CI-CD pipeline basics

### First Contributions
- [ ] Fix a typo or update documentation
- [ ] Make a small UI tweak or bug fix
- [ ] Write a test for existing code
- [ ] Go through code review process
- [ ] Merge your first PR! ðŸŽ‰

### Team Integration
- [ ] Attend team meetings/standups
- [ ] Understand team rituals and ceremonies
- [ ] Learn communication norms
- [ ] Identify who to ask for different types of help

### Using AI for Week 1
**AI Config**: `first_week_developer.md` or `onboarding_buddy.rules`

**Ask about:**
- Architecture and design patterns
- "How does [feature] work?"
- "Where is [functionality] implemented?"
- "Explain this code section to me"
- "What are the testing patterns here?"

## Week 2: Gaining Confidence

### Deeper Understanding
- [ ] Understand data models and database schema
- [ ] Learn API structure and patterns
- [ ] Understand state management approach
- [ ] Know authentication/authorization flows
- [ ] Explore error handling patterns

### Expanding Skills
- [ ] Take on a well-defined starter task
- [ ] Pair program with team member
- [ ] Participate in code review
- [ ] Debug an issue (with help)
- [ ] Write comprehensive tests for your changes

### Team Contribution
- [ ] Ask questions in team channels
- [ ] Share something you learned
- [ ] Help document something unclear
- [ ] Provide fresh perspective in discussions

### Using AI for Week 2
**Try pair programming config** if available

**Ask about:**
- "Help me implement [feature]"
- "Review my approach to [problem]"
- "What's the best way to test this?"
- "Are there edge cases I'm missing?"

## Week 3-4: Building Independence

### Independent Work
- [ ] Take on features end-to-end
- [ ] Estimate your own tasks
- [ ] Make architectural decisions (small scope)
- [ ] Debug issues more independently
- [ ] Write features with comprehensive tests

### Code Quality
- [ ] Follow team coding standards
- [ ] Write clean, readable code
- [ ] Provide thoughtful code reviews
- [ ] Refactor with confidence
- [ ] Consider performance and scalability

### Knowledge Sharing
- [ ] Document something you learned
- [ ] Help another new team member
- [ ] Share a useful resource with the team
- [ ] Ask advanced questions

### Using AI for Week 3-4
**Consider if you're ready for standard configs**

**Ask about:**
- "What's the best architecture for [feature]?"
- "How does this compare to [alternative approach]?"
- "Review my implementation approach"
- "Help me refactor this code"

## Month 2: Full Team Member

### Autonomous Contribution
- [ ] Work on regular team tasks
- [ ] Participate fully in sprint planning
- [ ] Provide meaningful code reviews
- [ ] Help with technical decisions
- [ ] Mentor newer team members

### System Knowledge
- [ ] Understand multiple areas of the codebase
- [ ] Know how pieces integrate
- [ ] Familiar with common bugs and gotchas
- [ ] Understand deployment and operations
- [ ] Know incident response procedures

### Team Integration
- [ ] Fully integrated into team workflow
- [ ] Contributing to team discussions
- [ ] Taking ownership of areas
- [ ] Comfortable asking for help when needed
- [ ] Comfortable saying no to scope creep

### Using AI at Month 2+
**Switch to standard team config**: `global_development.md`

**Ask about:**
- Complex architectural questions
- "Help me design [new system]"
- "What are the tradeoffs of [approaches]?"
- "Review my technical design"

## Month 3+: Experienced Team Member

### Leadership
- [ ] Lead features or projects
- [ ] Mentor new team members
- [ ] Contribute to technical decisions
- [ ] Improve team processes
- [ ] Share knowledge through talks/docs

### Expertise
- [ ] Deep knowledge in several areas
- [ ] Go-to person for certain topics
- [ ] Contribute to architecture decisions
- [ ] Identify and address technical debt

## Continuous Throughout

### Communication
- [ ] Daily: Participate in standups
- [ ] Weekly: Update task progress
- [ ] Ongoing: Ask questions when stuck
- [ ] Regular: Share wins and learnings
- [ ] Frequent: Seek and provide feedback

### Learning
- [ ] Read relevant documentation
- [ ] Explore related codebases/repos
- [ ] Learn from code reviews
- [ ] Study team's tech stack
- [ ] Keep AI configs up to date

### Well-being
- [ ] Take breaks
- [ ] Don't be afraid to ask for help
- [ ] Celebrate small wins
- [ ] Maintain work-life balance
- [ ] Remember: everyone was new once!

## Using This Checklist with AI

**Track progress**: Ask AI to help you track what you've completed
**Get guidance**: Ask AI to explain items you don't understand
**Plan your day**: Ask AI to help prioritize what to focus on
**Reflect**: Ask AI to help you review what you've learned

## Remember

- **Everyone's timeline is different** - Don't stress if you're faster/slower
- **It's okay to not know things** - That's why you're onboarding!
- **Ask questions** - The "dumb" question is the one not asked
- **Mistakes are learning** - Especially in your first weeks
- **AI is your assistant** - Use it to learn and accelerate, not replace thinking
